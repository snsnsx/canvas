# Онлайн-доска с совместной работой в реальном времени

Интерактивная HTML5-доска для совместного рисования: несколько человек одновременно
рисуют на бесконечном по горизонтали холсте, видят штрихи друг друга мгновенно и могут
созвониться голосом прямо в браузере.

Бэкенд — лёгкий сервер на **FastAPI** (синхронизация по WebSocket, состояние сохраняется
на диск). Фронтенд — ваниль JS без сборки. Проект рассчитан на запуск в **Google Colab**
с публикацией ссылки через **ngrok**.

## Возможности

- Рисование пером и маркером, ластик, выбор цвета и толщины
- Перемещение объектов, вставка изображений, экспорт в PNG
- Фон: без сетки / клетка / точки / линейка
- Отмена / повтор, очистка доски
- Голосовые звонки в браузере (WebRTC mesh, P2P-аудио; WebSocket используется только для сигналинга)
- Несколько независимых комнат: добавьте хэш к ссылке, например `…ngrok-free.app/#room1`
- Состояние каждой доски сохраняется на сервере в `boards/<room>.json`

## Расположение toolbar

Toolbar выполнен как плавающая command-панель поверх холста. Контролы сгруппированы по
сценариям: цвета пера, маркер, инструменты, толщина, фон, файлы, история и статус
участников онлайн. На узких экранах панель остаётся компактной и прокручивается
горизонтально.

## Быстрый старт в Google Colab (клонирование репозитория)

Сценарий: **Colab → клонирование репозитория → запуск со своим токеном ngrok.**

### 1. Сделайте свой форк (один раз)

Нажмите **Fork** вверху страницы репозитория на GitHub, чтобы получить копию в своём аккаунте.

### 2. Получите токен ngrok

Зарегистрируйтесь на <https://ngrok.com> и скопируйте authtoken со страницы
<https://dashboard.ngrok.com/get-started/your-authtoken>.

### 3. Добавьте токен в секреты Colab (рекомендуется)

В Colab слева откройте панель **🔑 Secrets** → **Add new secret**:

- **Name:** `NGROK_AUTHTOKEN`
- **Value:** ваш токен
- Включите тумблер **Notebook access**

Так токен не попадёт ни в код, ни в сохранённый вывод ноутбука. Если секрет не задан,
ноутбук безопасно спросит токен через скрытый ввод (`getpass`).

### 4. Откройте и запустите ноутбук

Откройте [`run_colab.ipynb`](run_colab.ipynb) в Colab одним из способов:

- В Colab: **File → Open notebook → GitHub**, вставьте URL своего форка и выберите `run_colab.ipynb`; **или**
- Замените в адресной строке `github.com` на `colab.research.google.com/github` для прямого открытия.

В первой ячейке укажите URL своего форка:

```python
REPO_URL = "https://github.com/snsnsx/canvas.git"
```

Затем выполните ячейки по порядку (**Runtime → Run all**). В конце появится публичная ссылка
вида `https://xxxx.ngrok-free.app` — поделитесь ею с участниками. При первом открытии страницы
ngrok нажмите **Visit Site**. Чтобы открыть отдельную доску, добавьте к ссылке хэш: `…/#room1`.

> На бесплатном плане ngrok одновременно доступен один туннель. Если запускаете повторно и
> видите ошибку «прежний туннель ещё закрывается» — выполните **Runtime → Restart session**.

## Запуск локально (без Colab)

```bash
git clone https://github.com/snsnsx/canvas.git
cd canvas
pip install -r requirements.txt
python app.py            # сервер на http://127.0.0.1:8000
```

Откройте <http://127.0.0.1:8000> в браузере. Туннель ngrok локально не нужен — он требуется
только для публичного доступа из Colab. Голосовые звонки браузер разрешит на `localhost` и на
HTTPS-домене (ngrok даёт HTTPS); по обычному HTTP на удалённом адресе доступ к микрофону
может быть заблокирован.

## Безопасность токена

- **Никогда не коммитьте свой ngrok-токен.** Он лежит в секретах Colab / переменной окружения,
  а `boards/` и `*.env` исключены через `.gitignore`.
- Если токен когда-либо попадал в код или в общий доступ — **отзовите и перевыпустите его** на
  <https://dashboard.ngrok.com/get-started/your-authtoken>. Токен даёт доступ к созданию туннелей
  от имени вашего аккаунта.
- Публичная ссылка ngrok открыта всем, у кого она есть. Не публикуйте её там, где не нужен
  доступ посторонних; останавливайте сессию, когда закончили.

## Как это работает

Сервер отдаёт `index.html` и JS-модули как статику и держит по одному состоянию `BoardState`
на каждую комнату. Клиенты подключаются по `ws://…/ws/<room>` и обмениваются операциями
(`beginStroke`, `appendPoints`, `moveObject`, `addImage`, `changeGrid`, `clearBoard`, undo/redo).
Сервер применяет операции к состоянию, ретранслирует их остальным участникам комнаты и с
дебаунсом сохраняет доску в `boards/<room>.json`. Сообщения голосового сигналинга (`rtc-*`)
сервер пересылает без изменения состояния — само аудио идёт напрямую между браузерами.
